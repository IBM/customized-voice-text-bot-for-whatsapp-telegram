Customized Voice and Text Chatbot fully integrated to database (Cloudant and COS) using Watson Services (STT, WA, TTS) deployed to IBM Cloud using Code Engine.

Flask

Flask is a popular web microframework that can be used for creating web applications. The other popular web framework for Python is Django, but it has many more dependencies than Flask. Flask is pretty simple to work with, and it allows for many design and customization choices from the developer, so it is good for quickly building web apps, as we have done with this chat bot. Watson Assistant can currently only communicate using text messages (in other words, strings), so in order to enable the use of audio messages with the chat bot, an application is needed to process audio messages. Thus, this Flask application is the backbone of the chatbot. Messages are forwarded through Twilio to the Flask application, which then processes each message differently, depending on whether the message is audio or text. The application will also eventually access a credit scoring machine learning model, so having a central application is good for coordinating all of the different services that go into this chat bot.

In src/whatsapp.py, Flask is used to create specific this RESTful API endpoint:

  • POST /chatbot-message
    • description: sending a message to the chatbot
    • returns: A Twilio MessagingResponse
      • this will have a textual body (text message) or an audio file attached (audio message)
